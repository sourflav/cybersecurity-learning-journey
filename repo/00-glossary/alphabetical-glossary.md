# Alphabetical Glossary

Technical terms collected during my cybersecurity learning journey.

## A
- **account** =
  Identity used to access a system or a service.
- **api** =
  Application Programming Interface. A set of rules, definitions, and tools that allows different software applications or components to communicate with each other.
- **authentication** =
  Process of verifying the identity of a user, device, or system. It is used to confirm that someone or something is really who they claim to be.


## B

## C
- **client** =
  A computer, device, or software application that sends requests to a server in order to access data, services, or resources. In a network, the client usually depends on the server to receive a response.

## D
- **device** =
  A physical piece of equipment used for a specific purpose. In computing, a device is usually a hardware component such as a computer, phone, keyboard, printer, or router.
- **directory** =
  A location in a file system used to organize and contain files and other directories. In computing, directory is another word for folder.

## E

## F
- **file** =
  A collection of data stored on a computer under a specific name. A file can contain text, images, code, audio, video, or other types of information.
- **folder** =
  A container used to organize and store files on a computer. A folder can also contain other folders, allowing information to be arranged in a structured way.
## G
- **gateway** =
  A device or network point that connects one network to another. In many local networks, the gateway is the router that allows devices to reach the internet or other external networks.
- **git** =
  A distributed version control system used to track changes in files over time. It helps users manage project history, compare revisions, and collaborate on the same project more effectively.
- **github** =
  A web based platform used to host, manage, and share software projects and other files tracked with Git. It is commonly used for version control, collaboration, documentation, and project organization.

## H
- **hub** =
  A central point used to connect multiple devices or components. In networking, a hub is a basic hardware device that sends incoming data to all connected devices without filtering or directing it intelligently.

## I
- **internet** =
  The global system of interconnected computer networks that allows devices around the world to communicate and exchange data. It supports services such as websites, email, file sharing, and online applications.
- **ip**
  Internet Protocol. A set of rules used to identify devices and route data across a network. It also refers to the unique address assigned to a device so it can send and receive information.
- **issue** =
  A problem, error, or situation that needs attention or resolution. In computing and technical work, an issue can refer to a bug, malfunction, limitation, or unexpected behavior in a system or project.

## K

## L
- **lan** =
  Local Area Network. A network that connects devices within a limited area, such as a home, office, school, or laboratory.

## M
- **markdown** =
  A lightweight markup language used to format plain text in a simple and readable way. It is commonly used to create documentation, notes, and README files on platforms like GitHub.
- **markup** =
  A system of symbols or tags added to text to define its structure, formatting, or presentation. In computing, markup is commonly used in languages like HTML and Markdown to organize and display content.

## N
- **net** =
  A shortened form of network or internet, depending on the context. In computing, it may refer to connected systems, online communication, or technologies related to networking.
- **network** =
  A group of connected devices that can communicate with each other and share data or resources. In computing, a network can be as small as a home connection or as large as the internet.
## O
- **os** =
  Operating System. The main software that manages a computer’s hardware, software resources, files, memory, and user interactions. It provides the environment in which applications run.

## P
- **path** =
  The location of a file or folder within a file system. A path shows the route used to reach a specific item on a computer, starting from a main directory or root.
- **pipeline** =
  A sequence of connected steps used to process data, perform tasks, or move work from one stage to another. In computing, a pipeline often describes an automated workflow where the output of one step becomes the input of the next.
- **policy** =
  A set of rules, guidelines, or principles that define how actions, decisions, or behavior should be managed. In computing and cybersecurity, a policy often establishes security requirements, access rules, or operational standards.
- **pull** =
  An action used to download and update local files with changes from a remote repository. In Git, pull is commonly used to bring the latest version of a project from GitHub or another remote source to a local machine.

## Q

## R
- **redundancy** =
  The duplication of components, data, or systems to increase reliability and reduce the risk of failure. In computing and cybersecurity, redundancy is often used to ensure that a service or resource remains available if one part stops working.
- **router** =
  A network device that forwards data between different networks. It is commonly used to connect a local network to the internet and to direct traffic to the correct destination.

## S
- **server** =
  A computer or software system that provides data, services, or resources to other devices or programs over a network. A server usually responds to requests from clients.
- **system** =
  A set of connected parts that work together to perform a function. In computing, a system usually refers to a computer, its operating system, software, hardware, and related components working together.

## T
- **task** =
  A specific piece of work or activity that needs to be completed. In computing, a task can refer to a process, operation, job, or action performed by a user, system, or program.

## U
- **user** =
  A person or account that interacts with a computer, system, application, or service. In computing, a user can have specific permissions, settings, and access rights.
